git学习笔记

发表于 2018-05-21 | 分类于 hexo
记录git学习情况

转载请申明出处，本文内容是向廖雪峰的官方网站学习的：https://www.liaoxuefeng.com

定义

GIT:分布式版本管理系统

commit_id:版本号

git命令

  1. git init ：初始化一个Git仓库
  2. git add ：文件添加
  3. git commit -m “” ：文件提交
  4. git status ：查看状态
  5. git diff ：查看difference
  6. git log ：显示最近到最远的提交日志
  7. git log – pretty = oneline ：提交日志一行行显示
  8. git reset ：回退到上一版本
  9. git reset –hard HEAD ：HEAD 指当前版本
  10. git reset –hard commit_id ：指定回到未来的某个版本
  11. git reflog ：显示做的每一次命令

工作区和暂存区

  • 工作区：Working Directory
  • 版本库：Repository
  • 暂存区：stage（或者叫index）
  • Git 自动创建的第一个分支：master
  • master 的一个指针：HEAD
  • git add把添加的文件提交到暂存区
  • git commit -m “” 把暂存区的所有文件全部提交到分支master

撤销修改

  1. git checkout –file：让file文件回到最近的一次git commit 或git add时的状态 ( 撤销工作区的修改 )(–很重要，git checkout 是切换到另一个分支的意思)
  2. cat file：查看file的内容
  3. git reset HEAD file：撤销暂存区的修改

删除文件

  • 删除工作区的文件：rm file
  • 删除版本库中的文件：git rm file , git commit -m “”
  • 删错了工作区的文件需要恢复：git checkout – file (实际上是用版本库里的版本替换工作区的版本)

远程仓库

  1. GitHub：提供Git仓库托管服务，只要注册一个GitHub账号，就可以免费获得Git远程仓库
  2. git push：把当前分支master推送到远程
  3. git push -u origin master：第一次推送时加上-u参数，Git会把本地的master分支内容推送到远程新的master分支，还会把master分支和远程的master分支关联起来，在以后的推送或拉取时可以简化命令，以后推送时都只需要用git push origin master。
  4. git remote add origin git @ server：关联一个远程仓库
  5. git clone：命令克隆
  6. 要克隆一个远程仓库，首先需要知道远程仓库的地址，然后使用
  7. Git支持多种协议，包括https，但是通过ssh支持的原生git协议速度最快

分支管理

创建与合并分支

  1. 创建并切换到dev分支：git checkout -b dev
  2. git checkout -b：创建并切换
  3. git branch：查看当前分支
  4. git checkout master：切换到master分支
  5. git merge dev：合并dev分支到当前分支
  6. Fast-forward：快进模式
  7. git branch -d dev：删除dev分支
  8. git branch dev：创建dev分支

解决冲突

git log – graph：查看分支合并图

分支管理策略

  • 一个团队可以创建多个分支工作，需要发布版本时再合并到master分支上并发布

  • Fast forward模式下删除分支后会丢掉分支信息

  • 要强制禁用快进模式需要使用–no-ff参数

  • 例如创建了dev分支，准备合并dev分支时：

    git merge –no-ff -m “” dev

    (-m “” ：本次合并创建一个新的commit，””里是commi的描述)

Bug分支

首先确定在哪个分支上修复Bug，如需在master分支上修复，就从master分支创建临时分支，创建后修复Bug，提交，修复完成后，切换到master分支，并完成合并，最后删除临时分支

例如：当前工作在dev分支上，在master分支上创建临时分支issue-101，修复readme.txt

  1. git stash ：保存当前工作
  2. git checkout master ：切换到分支master
  3. git checkout -b issue-101 ：创建并切换到分支 issue-101
  4. git add readme.txt ：修复Bug后提交到暂存区
  5. git commit -m “fix bug 101” ：提交到版本库
  6. git checkout master ：切换到分支master
  7. git merge –no-ff -m “merged bug fix 101” issue-101 ：合并issue-101分支到master分支
  8. git branch -d issue-101 ：删除分支issue-101
  9. git checkout dev ：切换到分支dev
  10. git status ：查看状态 (工作区)
  11. git stash list ：查看工作现场在哪里
  12. git stash apply ：恢复
  13. git stash drop ：删除
  14. git stash pop ：恢复并删除stash内容

Feature 分支

如果要开发一个新功能，最好新建一个分支

如果要丢弃一个没有被合并过的分支，可以通过 git branch -D file 强行删除file

多人协作

  1. 查看远程信息，用 git remote -v
  2. 本地新建的分支如果不推送到远程，对其他人就是不可见的
  3. 从本地推送分支，使用 git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交
  4. 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致
  5. 建立本地分支和远程分支的连接，使用git branch –set - upstream branch-name origin/branch-name
  6. 从远程抓取分支，使用git pull，如果有冲突，先处理冲突

标签管理

创建标签

标签是静止的，可以取一个简易的名字方便查找，例如 v1.0

  1. 打标签：先切换到需要打标签的分支上，然后git tag name
  2. git tag ：查看所有标签
  3. git tag name ：新建一个标签，默认为HEAD，也可以指定一个commit id
  4. git tag -a tagname -m “” ：指定标签信息 (-a：指定标签名；-m：指定说明文字)
  5. git tag -s tagname -m “” ：用PGP签名标签
  6. git show tagname ：查看标签信息

例如:因为默认标签是打在最新提交的commit上的，因此如果要在这周一打标签，但是标签需要打在上周三提交的commit id 上，需要找到历史提交的commit id，然后打标签

git log – pretty = oneline – abbrev - commit

如找到的commit id是 6224973

git tag v0.9 6224973

操作标签

  1. git tag -d v0.1 ：删除本地标签v0.1

  2. git push origin v0.1 ：推送v0.1到远程

  3. git push origin –tags ：一次性推送全部尚未推送到远程的本地标签

  4. 如果标签已经推送到远程，先删除本地标签，然后从远程删除 ：

    • git tag -d v0.9
    • git push origin : refs/tags/v0.9
